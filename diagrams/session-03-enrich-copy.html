The Core Principle
Clay is an API waterfall with a UI on top. This session shows you the waterfall itself. LeadMagic finds the email and verifies it. AI Ark steps in when LeadMagic misses. Exa researches every lead for intent signals. Claude scores the context against your firm's value proposition and writes copy calibrated to what it found. No platform fee. No credit system. You own every step.

Step 01 · Email Waterfall
LeadMagic — Primary Email Finder
LeadMagic API Built-in Verification

Hits the LeadMagic /profile-find endpoint by LinkedIn URL. Returns email, email status, company name, domain, and employee count. Verification is native — no separate call needed. If LeadMagic finds a valid or catch-all email, skip AI Ark entirely.

email email_status company_name company_domain employee_count
Step 01 (Fallback) · Email Waterfall
AI Ark — Email Fallback
AI Ark API Catch-All Aware
↺ Only runs if LeadMagic returned no email

AI Ark's email finder handles catch-all domains and bounce detection natively. Confidence scores map directly to email status: high→valid, medium→catch_all, low→invalid. If neither API finds an email, the lead continues to Exa with email_status: not_found — enrichment still runs.

email email_source email_status
Step 02 · Research (Every Lead)
Exa — Firmographics + Intent Signals
Exa MCP Cached Every 5 Calls

Runs on every lead regardless of email result. people_search_exa queries the lead's name + company, surfacing recent posts, hiring signals, tech stack, and any pain signals that match your firm's ICP. Results are cached to /tmp/exa-cache-{SHEET_ID}.json after every 5 calls — if the run is interrupted, it resumes without repeating completed searches.

Fills gaps in: company_name, domain, employee_count, tech_stack
exa_context exa_intent_signals tech_stack
Step 03 · In-Context Scoring
Exa Context Scoring — Claude In-Context
Claude Code Free · No API

Claude compares each lead's Exa results against the Firm Context block in the skill file — your value proposition, ICP definition, and top pain signals. If the block is empty, it falls back to reading CLAUDE.md. No API call needed; this is in-context reasoning.

Writes to: “Enriched & Verified” tab — then pauses for approval
context_score matched_signals
High — signal match Medium — ICP confirmed Low — generic context
Step 04 · Copy Generation
Copy Generation — Pilot 10 → Approve → Full Batch
Claude Code 3 Copy Paths

Valid and catch-all emails only. Claude generates a cold email (Subject, Hook, Body, Value Proposition, CTA, P.S.) and a LinkedIn DM per lead using one of three paths based on context score. High-context leads get signal-specific copy. Medium gets role + company personalization. Low gets clean ICP-focused copy with no forced context.

Pilot gate on first 10: subject + hook displayed per lead. Approval required before full batch runs.
