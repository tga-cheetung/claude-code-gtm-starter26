The Core Principle
Email enrichment costs money — per contact, per API call. If you enrich your entire list of raw LinkedIn engagers, you'll pay for people who are agency owners, ghostwriters, and service providers who will never buy. Session 2 is the gate that runs before any paid enrichment. Claude Code reads the data, applies the filter, scores each lead, and checks the CRM — all in a single conversation, all for free.

Step 01
Headline Filter
Claude Code Free · No API

Regex-based pass/fail against the LinkedIn headline. Requires at least one founder signal (CEO, Co-founder, Founder). Excludes non-ICP roles: ghostwriters, SDRs, recruiters, account managers, LinkedIn ad specialists.

Writes to: “Headline Filtered” tab
icp_headline_pass headline_filter_reason has_saas_signal
Step 02
AI ICP Scoring
Claude Code Free · In-Context

Claude reads each headline and scores ICP fit 0–10 against the target ICP. Catches nuance keywords can't — scores a “Founder | GTM Agency” as 2/10 because they're a service provider, not a product founder. Pilots first 10 rows for human review before scoring the rest.

Writes to: “ICP Scored” tab
score score_label rationale is_saas_founder
Step 03
RevyOps Dedup Check
curl Free · Own DB

Checks every LinkedIn URL against the RevyOps CRM master list via REST API. Prevents re-enriching leads already in the database or re-targeting contacts mid-sequence.

Writes to: “Ready for Enrichment” tab
is_duplicate revyops_status revyops_contact_id
